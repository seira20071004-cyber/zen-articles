# 初めてのアプリ開発のために環境構築をしてみました！

## 1. はじめに
今回は、アプリ開発をする上でまずやらなくてはいけない環境構築をしたのでまとめてみました！
思ったより大変で苦労したので、簡単に振り返っておきたいと思います✨。

### なぜアプリ開発をしようと思ったのか
私がアプリ開発をしようと思い立った理由は、大きく2つあります！

1つ目は、**IT関連の仕事でアプリ開発は必ずためになると思ったから**です。
私はインフラ系の仕事やSIerに興味があり、開発自体はあまり使わないのでは？と思っていました。しかし、アプリ開発をすることによって以下のようなメリットがあると考えました。

* **「技術のつながり」がわかる**
  画像のデザイン（フロントエンド）、データを保存する仕組み（バックエンド）、アプリを動かすサーバー（インフラ）など、IT業界で必須となる要素がどう連携しているかを体感できる。
* **IT資格の勉強が楽になる**
  技術同士のつながりがわかるだけでなく、抽象的な専門用語（「API」、「HTTP通信」など）が「アプリでいうこれのことか！」と理解しやすくなります。

2つ目は、資格勉強や学校の試験勉強をしていく中で、逆算して計画を立てるといったことが面倒になってきたので、**自動逆算計画アプリを作ってしまおうと考えたから**です。
私と同じような悩みを持っている人も少なくないと思うので、完成したら最近始めたGitHubやInstagramで共有できたらいいなと考えています。

## 2. 私の開発環境
開発環境はパソコンによってエラーになる原因が異なるため、私のベースとなる環境を載せておきます。

* **PC:** Windows 11
* **プログラミング経験:** Pythonの基本文法がわかる程度

## 3. アプリ開発の相棒に「Cursor×Flutter」を選んだ理由
アプリ開発は大きく「ノーコード開発」と「フルコード開発」があります。

* **ノーコード開発:** コードを書かずに直感的なUI操作で開発。
* **フルコード開発:** すべてコードで書き、完全な自由度で開発可能。

これらを踏まえて、私は「スキルをつけること」が目的なので、しっかり操作できるフルコード開発を選びました。（ローコード開発と呼ばれる2つの中間のような方法もありますが、まずはシンプルにフルコードを選びました）

### Flutterを選んだ理由
1つのコードでiOSとAndroidの両方に対応できる「クロスプラットフォーム開発」であり、UIの作成がとても見やすく、開発効率が良いことから選びました。

### Cursorを選んだ理由
今話題の「AIが組み込まれたエディタ」ということで、未経験の私でもAIを相棒にすれば、エラーを乗り越えていけるのではないかと思い採用しました。

## 4. 環境構築の流れ
環境構築の手順は以下の通りです。それぞれ公式ページからダウンロードや登録を行いました。

1. **GitHubのアカウント登録**（※すでに持っている方は不要です）
   [GitHub 公式サイト](https://github.com/)
2. **Cursor（エディタ）のインストール**
   [Cursor 公式ダウンロードページ](https://www.cursor.com/)
3. **Flutter SDKのインストール**（お使いのPCのOSに合わせて手順に従います）
   [Flutter 公式インストールガイド](https://docs.flutter.dev/get-started/install)
4. **環境構築・連携の完了**

基本的には、最初に登録したGitHubアカウント（またはGoogleアカウント）を使ってCursorなどの各ツールにログイン・連携していけば、スムーズに初期設定が進みます！

### Cursorを日本語化する方法
Cursorは初期状態だと画面が英語なので、使いやすくするために日本語化の拡張機能を入れましょう。（※CursorはVS Codeというエディタをベースに作られているため、VS Codeと同じ拡張機能が使えます！）

手順は以下の通りです。
1. `Ctrl` + `Shift` + `X` キー（Macは `Cmd` + `Shift` + `X`）を同時に押して、拡張機能（Extensions）画面を開く。
2. 左上の検索欄に「**Japanese**」と入力する。
3. 一番上に出てくる地球マークの「Japanese Language Pack for Visual Studio Code」の「**Install**」ボタンを押す。
4. `Ctrl` + `Shift` + `P` キー（Macは `Cmd` + `Shift` + `P`）を同時に押して、コマンドパレットを開く。
5. 「**display**」と入力し、絞り込まれた項目から「**Configure Display Language**」を選択してEnter。
6. 言語の選択肢が出てくるので「**ja**」を選択する。
7. 右下に出てくるポップアップの「**Restart**（再起動）」ボタンを押して、Cursorを再起動すれば日本語化完了です！

### 🛠️ 詰まったポイント・アドバイス
* **インストーラーの設定はデフォルトでOK：** インストール中の選択肢は、基本的に変更せずそのまま「Next」で進めていって大丈夫です。
* **英語での設定に焦らなくても大丈夫：** 途中で英語の設定画面がいくつか出てきて身構えてしまいますが、スキップできるところはスキップして構いません。
* **GitHub連携時のリポジトリ選択：** GitHubと連携する際のアクセス権限の選択では、基本的に「All repositories（すべてのリポジトリ）」を選択したまま進めて問題ありませんでした。

もし手順の中で不明な点やエラーが出たところがあれば、ぜひコメントで教えてください！

## 5. 実際に環境構築を終えてみて感じたこと
