# zenn-content

Zennの記事と本を管理するためのリポジトリです。
GitHubへのプッシュ（mainブランチへのマージ）をトリガーに、Zenn（zenn.dev）へ自動同期・公開されるように設定しています。

## 🛠️ 開発環境の準備

この記事の作成やローカルプレビューを行うには、以下の環境が必要です。

### 1. 前提条件のインストール
- **Node.js** (LTS推奨版)
  - `npm` および `npx` コマンドを使用するために必要です。未インストールの場合は公式サイトからダウンロードしてください。
- **Visual Studio Code (VS Code)**
  - 記事の執筆およびターミナル操作に使用します。

### 2. リポジトリのクローンとセットアップ
1. 本リポジトリをローカル環境にクローンし、VS Codeで開きます。
2. VS Codeのメニューから **「ターミナル」＞「新しいターミナル」** を開きます。
3. 以下のコマンドを実行し、必要な依存パッケージ（`zenn-cli`など）を一括インストールします。
   ```bash
   npm install

## 🚀使い方

### 1.ローカルでのプレビュー（推奨）
Zenn特有のメッセージボックス、リンクカード、実際のブログデザインを正確に確認するために、ローカルプレビューを使用します。
```Bash
npx zenn preview
```
実行後、ブラウザで`http://localhost:8000`にアクセスすると、Zennのサイトと全く同じ見た目で記事をプレビューできます。

### 2.記事の新規作成
新しい記事を書くときは、以下のコマンドを使います。
```Bash
npx zenn new:article
```
※作成された`md`ファイルの冒頭にある`publishd:false`を`True`に書き換えることで、公開状態をコントロールできます。

## 🔒運用について（セキュリティ面）
本リポジトリの書き込み権限は管理者（アカウントオーナー）のみに制限されています。第三者が閲覧・クローンすることは可能ですが、内容が勝手に書き換えられたり、意図しない記事が公開されることはありません。
